## The Pattern — Fractal Logic, Simply Explained

A fractal is a pattern that repeats at every scale. A tree branch looks like the whole tree. A coastline looks the same from space and from your feet.

**Fractal logic** means: the same rule that works at the smallest scale works at every scale. You don't need different rules for different sizes. You need *one good rule*.

In a Personal Knowledge Management (PKM) system, fractal logic looks like this:

| Scale | The Rule |
|-------|----------|
| **One note** | Has a unique ID, frontmatter, links to related notes |
| **One folder** | Has an index, follows naming conventions, links to related folders |
| **One agent** | Has a contract, a journal, SOPs, links to related agents |
| **The whole system** | Has the same structure — indexed, linked, self-documenting |

**One rule. Every scale.**

That's it. That's the fractal secret. You don't need complexity. You need *recursive consistency*.
